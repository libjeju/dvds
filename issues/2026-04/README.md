# 제주대학교 중앙도서관 추천DVD

GitHub Pages에 바로 게시할 수 있는 정적 웹사이트 저장소입니다.

## 이번 호

* 기간: 2026년 7–8월
* 주제: **당신에게 뜨거운 여름을**
* 구성: 뉴스레터, 카드뉴스, 웹메일 발송용 본문 복사 도구

## 저장소 구조

```text
recommended-books/
├── index.html                         # 추천DVD 아카이브 홈
├── newsletter/2026-07-08/index.html  # 반응형 뉴스레터 v6
├── cardnews/2026-07-08/index.html    # 카드뉴스 v8 웹 갤러리
├── cardnews/2026-07-08/png/           # SNS 게시용 PNG 9장
├── email/2026-07-08/index.html       # 제목·서식 본문 복사 도구
├── assets/covers/2026-07-08/          # 표지 이미지 자리
├── tools/                             # 로컬 점검·카드뉴스 출력 도구
└── .github/workflows/pages.yml        # GitHub Pages 자동 배포
```

## 처음 게시하는 방법

1. GitHub 조직 또는 도서관 공식 계정에서 공개 저장소 `recommended-books`를 만듭니다.
2. 이 압축파일의 **내용물 전체**를 저장소 최상위에 업로드합니다. 압축파일 자체를 올리지 않습니다.
3. 저장소에서 **Settings → Pages → Build and deployment**로 이동합니다.
4. Source에서 **Deploy from a branch**, Branch에서 **main**, Folder에서 \*\*/(root)\*\*를 선택하고 저장합니다.
5. Pages 화면에 표시된 주소를 엽니다. 일반적인 주소 형식은 `https://계정명.github.io/recommended-books/`입니다.
6. 발송용 도구는 `.../email/2026-07-08/`에서 엽니다. 제목과 서식 본문을 복사해 제주대학교 웹메일에 붙여 넣습니다.

## 실제 메일 발송

* HTML 파일을 첨부하지 않습니다.
* 긴 PDF 또는 한 장짜리 긴 이미지를 본문 대신 첨부하지 않습니다.
* 웹메일에는 짧은 소개와 **전체 뉴스레터 보기** 링크를 넣습니다.
* 발송 전 본인, 동료, 스마트폰 계정으로 시험 발송합니다.
* 웹메일에서 서식이 깨지면 `일반 텍스트 복사`를 사용합니다.

## 표지 이미지 교체

현재 `assets/covers/2026-07-08/`에는 깨짐 방지를 위한 자체 제작 대체 이미지가 들어 있습니다. 실제 소장 판본의 표지를 사용할 경우 저작권·이용 조건을 확인한 뒤, 같은 파일명으로 덮어쓰면 뉴스레터와 카드뉴스에 함께 반영됩니다.

# 운영·발행 가이드

## 권장 운영 구조

1. 웹메일: 짧은 소개와 링크
2. GitHub Pages: 20권 전체 뉴스레터와 아카이브
3. 카드뉴스: 웹 갤러리와 PNG 게시물
4. 도서관 OPAC: 소장 위치와 대출 상태 확인

## 웹메일 시험 발송 체크리스트

- [ ] 제목에 기관명, 첫 발행, 주제가 포함되었는가
- [ ] 발신자명이 `제주대학교 중앙도서관`으로 보이는가
- [ ] 전체 뉴스레터 링크가 열리는가
- [ ] 카드뉴스 링크가 열리는가
- [ ] 모바일에서 본문이 한 화면 폭에 맞는가
- [ ] 이미지가 차단되어도 핵심 내용이 전달되는가
- [ ] 기간·장소·대상 정보가 확정되었는가
- [ ] 맞춤법과 도서명이 최종 교정되었는가
- [ ] 내부 대량 발송 승인을 받았는가

## PDF·PNG 첨부에 대한 원칙

PDF는 보존·인쇄용 보조 파일로는 사용할 수 있지만, 첫 화면에서 바로 읽기 어렵고 모바일에서 확대가 필요합니다. 긴 PNG/JPG는 글자가 작아지고 대체텍스트를 제공하기 어렵습니다. 따라서 메일의 주된 전달 방식으로는 사용하지 않고, 웹페이지 링크를 중심으로 운영합니다.
